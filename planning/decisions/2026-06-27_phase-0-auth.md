# Phase 0 Auth & Scaffolding Decisions

Decisions locked in the Phase 0 grilling session (2026-06-27).

## Sessions

**Decision:** Server-side sessions stored in PostgreSQL (`sessions` table).
Cookie holds an opaque 64-char hex token; the row holds `user_id + expires_at`.

**Why:** Logout actually works (delete the row). A lost device can be revoked.
JWTs can't be revoked without a server-side denylist ‚Äî which is just sessions again.

**Session duration:** 30-day sliding expiry. Each request resets `expires_at`.
Configurable via `SESSION_EXPIRE_HOURS` env var (default `720`).

## User creation

**Decision:** No `/register` endpoint. Geodude creates accounts via CLI:

```bash
uv run python cli.py create-user --email X --name Y --password Z
```

**Why:** Private family app ‚Äî a public registration endpoint is unnecessary attack
surface. Anyone who reaches the app over Tailscale should not be able to create
their own account.

## Auth endpoints (Phase 0)

- `POST /api/auth/login` ‚Äî rate-limited (`LOGIN_RATE_LIMIT`, default 10/minute)
- `POST /api/auth/logout` ‚Äî deletes session row server-side
- `POST /api/auth/change-password` ‚Äî authenticated; no forced-change flag
- `GET /api/auth/me` ‚Äî returns current user (used by the frontend auth guard)

## Password reset

Deferred. Geodude can reset a password by running `create-user` and deleting the
old account via `psql`, or a future `reset-password` CLI command.

## MFA

**Decision:** Seam only. `users.mfa_secret` is nullable. The auth service has:

```python
if user.mfa_secret:
    # verify TOTP here ‚Äî active when mfa_secret is populated (Phase 7)
    pass
```

No endpoint, no feature flag. Phase 7 switches it on by populating the column and
adding the TOTP endpoint.

## Rate limiting

**Decision:** `slowapi` on the login endpoint (in-process). Nginx-level rate
limiting can be layered on in Phase 7 as defense-in-depth.

**Why:** Works identically in dev and prod; the rate-limit behaviour is testable.

## Frontend

**Decision:** SvelteKit in SPA mode ‚Äî `adapter-static` with `fallback: 'index.html'`,
SSR disabled. All routing is client-side; FastAPI handles all data.

**Dev API:** Vite proxies `/api/*` ‚Üí `http://localhost:8000`. Frontend always uses
relative `/api/...` paths ‚Äî zero difference between dev and prod.

## Alembic

Started in Phase 0. Initial migration (`0001_initial`) covers `users` + `sessions`.
Phase 1 adds financial tables using the same tooling.

## CI/CD

Deferred to Phase 7. The `check` skill gates each phase manually; no cloud runner
needed for a solo project.

## Deployment target

Laptop first; secondary (always-on) device is a Phase 7 decision. Multi-arch image
concerns are deferred until the device is chosen.
